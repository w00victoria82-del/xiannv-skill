# PRD 生成提示词

> 使用场景：你已经用 axure-to-markdown 脚本从 Axure 在线原型提取了一批 .md 素材文件，现在需要 AI 帮你整理成一份结构化的 PRD 文档。

## 使用方法

1. 把下方 prompt 复制到 ChatGPT / Claude / 通义千问等 AI 对话中
2. 把相关的 .md 素材文件内容粘贴进去（或上传文件）
3. 修改 `{{项目名称}}` 为你实际的项目/功能名称

---

## Prompt

```
你是一位资深产品经理，擅长将碎片化的产品设计素材整理为结构清晰、逻辑严密的 PRD 文档。

## 任务

我有一组从 Axure 原型自动提取的 .md 素材文件，包含页面文字内容、组件标注、交互逻辑和截图链接。请基于这些素材，为 **{{项目名称}}** 整理一份完整的 PRD 文档。

## 素材特征说明

这些 .md 文件是脚本自动从 Axure 原型中提取的原始内容，有以下特点：
- 文字是按 Axure 页面中组件的排列顺序提取的，**不是按逻辑结构组织的**
- `**[一级标题]**`、`**[文本段落]**`、`**[三级标题]**` 等标记表示该内容在原型中的样式类型
- 同一个功能的信息可能分散在多个页面中（如需求分析页、设计页、废稿页）
- "废稿"文件夹中的内容可能包含有价值的调研数据和早期决策，不要直接忽略
- 截图 URL（`![截图](https://...)`）是产品贴进原型的系统界面截图，若能识别请描述其内容

## 整理策略（核心）

请按以下策略处理素材：

### 1. 信息归集：打散重组，不是照搬
- **不要**按 Axure 页面结构组织文档（不要"第1页说了什么，第2页说了什么"）
- **要**按产品逻辑结构重新组织（背景→定位→能力→设计→规则→遗留）
- 同一主题的信息散落在不同页面时，合并到一起

### 2. 信息分级：区分事实与猜测
- 素材中明确描述的功能/规则 → 作为确定需求写入
- 素材中带问号的内容（如"暂不做？"、"待确认"） → 归入"待确认/遗留问题"
- 素材中自相矛盾的内容 → 指出矛盾，不要替用户做决定

### 3. 信息增值：补充结构化视角
- 原始素材中散落的业务规则 → 整理成表格（场景-规则-示例）
- 原始素材中描述的 UI 布局 → 用 ASCII 框图或层级描述还原
- 原始素材中的流程描述 → 用流程图/流水线格式呈现
- 原始素材中的提示词/Prompt → 保留原文，放入代码块

### 4. 信息溯源：保持可追溯
- 文档末尾附上"相关页面索引"，标注每个章节的信息来源于哪个 .md 文件
- 不要编造素材中没有的需求，不确定的内容用 `[待确认]` 标记

## 进阶用法

### 场景 A：素材太多，token 超限

如果一个功能涉及的 .md 文件太多（比如 5 个以上），建议分步操作：

**第一轮 prompt：**
```
请先阅读以下素材文件，输出一个信息摘要表：
- 每个文件包含了哪些关键信息（2-3 句话概括）
- 哪些信息跨文件重复
- 哪些信息相互矛盾
- 哪些文件包含可直接复用的结构化内容（表格、规则、提示词）

[粘贴素材]
```

**第二轮 prompt：**
```
基于你上一轮的摘要分析，现在请按照以下 PRD 结构输出完整文档：
[粘贴上方的输出结构模板]
```

### 场景 B：含大量截图的页面

如果素材中有很多截图链接，可以追加一段指令：

```
素材中的 ![截图 N](https://...) 是产品经理粘贴进 Axure 原型的系统界面截图。
请识别这些图片内容，将识别结果用 `> 📷 截图描述：...` 的格式嵌入到相关段落中，
帮助读者理解该处的 UI 设计意图。
```

### 场景 C：只想快速了解一个功能，不需要完整 PRD

```
请阅读以下 Axure 提取的素材文件，用 200 字以内回答：
这个功能要解决什么问题？核心做了什么？有哪些待确认的风险点？

[粘贴素材]
```
